$ git add --all #tmb conocido como $ git add . #Especificando $ git add *.txt #(agrega todos los archivos con esa terminación).
git commit -m "mensaje" #Esto mueve definitivamente al repo los files q estaban en staging area
git push #sube el repo a GitHub
git pull #clona en tu repo local lo que estaba en GitHub
git clone <link>
git status #el estado de los archivos sin seguimiento o cambios dentro de los archivos que no han sido agregados y también los archivos que han sido eliminados, estos estados son:
* working directory: cambios que no están en el seguimiento o archivos que no han sido agregados para ser seguidos.
* staging area: archivos (o sus cambios) que serán agregados al seguimiento (en algunas ocaciones no queremos subir algunos archivos de inmediato y seleccionamos solo los que nos interesan).

git diff #muestra los cambios hechos en los archivos del repo
